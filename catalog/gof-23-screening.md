# Gang of Four 23-Pattern Screen

This screen evaluates whether each canonical Gang of Four pattern has a coherent Skillware mapping and records the strongest correspondence in the reviewed source set. **Screening does not imply detailed implementation, ecosystem frequency, or quality.** A candidate correspondence remains subject to fixed-path, participant-level evidence and a focused misuse discriminator.

| Pattern | 中文名 | Category | Source Intent | Skillware Carriers | Result | Screening Reasoning | False-Positive Risk | Detailed Sample |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| Abstract Factory | 抽象工厂模式 | Creational | Create related or dependent object families without naming concrete classes. | Host renderers; install-target adapters; manifest templates | Plausible | gstack Host renderers and ECC install-target adapters are candidates for compatible Host product families. | One generated Host file does not establish a compatible product family. | No |
| Builder | 建造者模式 | Creational | Separate complex construction from representation so one process creates different representations. | Generator Skill; construction schema; profile-specific writers | Strong | dot-skill stages material collection, analysis, schema-driven writing, and installation of generated Skillware. | One template expansion without staged construction or representation variation is not Builder. | No |
| Factory Method | 工厂方法模式 | Creational | Define a creation interface while concrete creators choose the produced product. | Creator Skill; family profiles; scaffolding scripts | Plausible | dot-skill family profiles are candidate concrete creators. | Static template selection without a creator contract is not Factory Method. | No |
| Prototype | 原型模式 | Creational | Create new objects by copying a prototypical instance. | Skill templates; profile seeds; controlled copy scripts | Plausible | dot-skill templates and profile seeds are candidate prototypes; controlled copy semantics require further validation. | Directory copying without identity and customization semantics is not Prototype. | No |
| Singleton | 单例模式 | Creational | Ensure one instance and provide a global access point. | Session-scoped service; profile configuration owner | Weak | No focused case demonstrates a unique runtime instance across Host and session boundaries. | A root file or preferred entry point does not prove runtime instance uniqueness. | No |
| Adapter | 适配器模式 | Structural | Convert one interface into another expected by clients. | Canonical Skill; Host binding; manifest; tool mapping | Strong | gstack translates canonical Skill behavior into Host-specific paths, metadata, and tool vocabulary. | Renaming or copying a file without semantic translation is not Adapter. | Yes |
| Bridge | 桥接模式 | Structural | Separate an abstraction from its implementation so both vary independently. | Canonical action vocabulary; Host resolver; tool mapping | Plausible | gstack's canonical actions and Host resolvers are a candidate separation of abstraction and implementation dimensions. | A single translation shim is Adapter unless both dimensions vary independently. | No |
| Composite | 组合模式 | Structural | Compose part-whole trees and treat individuals and compositions uniformly. | Parent Skill; child Skills; workflow manifest | Strong | OpenMontage's stage system is a candidate; the companion repository verifies a common component contract and acyclic part-whole structure. | A directory of unrelated Skills without a shared contract is not Composite. | Yes |
| Decorator | 装饰器模式 | Structural | Add responsibilities dynamically while preserving the component interface. | Wrapper Skill; activation wrapper; pre-hook; post-hook | Strong | Caveman activation wrappers are a candidate; the companion repository verifies contract-preserving responsibility layering. | A hook that changes the task contract or replaces core behavior is not Decorator. | Yes |
| Facade | 外观模式 | Structural | Provide a unified interface that makes a subsystem easier to use. | Entry Skill; root router; invocation policy | Strong | Superpowers using-superpowers and gstack's root router implement unified entry contracts over independently addressable specialist Skills. | A README or catalog supplies navigation, not an executable access contract. | Yes |
| Flyweight | 享元模式 | Structural | Share fine-grained objects through separated intrinsic and extrinsic state. | Shared references; shared templates; caches | Weak | Shared references alone do not demonstrate intrinsic state sharing and separated extrinsic state. | Ordinary file reuse does not establish the required state split. | No |
| Proxy | 代理模式 | Structural | Provide a compatible surrogate that controls access to another object. | Local daemon; MCP server; permission gateway | Plausible | gstack's browser daemon and EvoZeus dispatch paths are audit candidates; neither currently demonstrates a verified same-interface surrogate relation. | A gateway or adapter without a compatible surrogate interface is not Proxy. | No |
| Chain of Responsibility | 职责链模式 | Behavioral | Pass a request among potential handlers until one handles it. | Handler Skills; typed handoff records; workflow routes | Plausible | ECC and financial-services orchestration paths are candidates; each handler must independently decide to handle or forward the same request. | A pipeline where every stage runs is not Chain of Responsibility. | No |
| Command | 命令模式 | Behavioral | Encapsulate a request so invocation, logging, queuing, or undo can vary. | Command record; slash command; receiver Skill; audit record | Plausible | ECC command-to-Skill routes are candidates; invoker, request object, receiver, and audit or undo semantics require verification. | A shortcut without an encapsulated request and receiver relation is not Command. | No |
| Interpreter | 解释器模式 | Behavioral | Represent a language grammar and interpret sentences in that language. | Workflow grammar; parser; expression records | Plausible | OpenMontage's pipeline language and parser are candidates when grammar and expression participants are evidenced. | Model interpretation of prose without a grammar and expressions is not Interpreter. | No |
| Iterator | 迭代器模式 | Behavioral | Traverse aggregate elements without exposing the underlying representation. | Catalog traversal contract; paginated scanner | Weak | Catalog scans and ordinary loops provide insufficient evidence of an independent traversal contract. | A loop or recursive scan alone is not Iterator. | No |
| Mediator | 中介者模式 | Behavioral | Encapsulate peer interaction so colleagues do not refer to each other explicitly. | Coordinator Skill; parent Agent; coordination manifest | Strong | financial-services workflow orchestration and EvoZeus coordination are candidates; the companion repository verifies colleague isolation. | A God Skill that absorbs specialist work is not a sound Mediator. | Yes |
| Memento | 备忘录模式 | Behavioral | Capture internal state without breaking encapsulation for later restoration. | Release snapshot; checkpoint; caretaker harness | Strong | SkillOpt backups and EvoZeus release snapshots provide partial evidence of capture, caretaker ownership, and recovery. | A backup without an owned operational restore path is not necessarily Memento. | Yes |
| Observer | 观察者模式 | Behavioral | Notify dependent observers when a subject changes state. | Lifecycle event; hook; subscription record; event consumer | Strong | ECC hooks and OpenMontage lifecycle events are candidates; explicit subject, registration, and notification semantics require path validation. | Polling or an unregistered hook is not Observer. | Yes |
| State | 状态模式 | Behavioral | Alter behavior when internal state changes. | Persisted checkpoint; state record; transition manifest | Strong | SkillOpt stages and OpenMontage checkpoints provide candidate persisted state-dependent behavior. | Phase names or stateless routing do not establish State. | Yes |
| Strategy | 策略模式 | Behavioral | Define interchangeable algorithms behind a common interface. | Specialist Skills; selection policy; shared request contract | Strong | ui-ux-pro-max retrieval procedures and financial-analysis workflows provide candidate interchangeable strategy families. | Alternatives with incompatible result contracts are not Strategies. | Yes |
| Template Method | 模板方法模式 | Behavioral | Fix an algorithm skeleton while allowing selected steps to vary. | Workflow Skill; invariant skeleton; domain profiles | Strong | Superpowers workflows provide candidate invariant skeletons with bounded task-specific steps. | A freely reordered checklist or interchangeable whole algorithm is not Template Method. | Yes |
| Visitor | 访问者模式 | Behavioral | Separate operations from a stable object structure so new operations do not change elements. | Repository scanner; validator; heterogeneous element records | Plausible | Repository scanners and validators are candidates when operation dispatch over a stable heterogeneous element structure is evidenced. | A generic scan without element-specific dispatch is not Visitor. | No |

The current screening distribution is 11 Strong, 9 Plausible, and 3 Weak. Only the ten rows marked **Yes** have detailed GoF samples in this repository plan; the other thirteen do not claim one.
