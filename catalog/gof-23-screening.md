# Gang of Four 23-Pattern Screen

This screen evaluates whether each canonical Gang of Four pattern has a coherent Skillware mapping and records the strongest correspondence in the reviewed source set. **Screening does not imply detailed implementation, ecosystem frequency, or quality.** A candidate correspondence remains subject to fixed-path, participant-level evidence and a focused misuse discriminator.

| Pattern | 中文名 | Category | Source Intent | Skillware Carriers | Result | Screening Reasoning | False-Positive Risk | Detailed Sample |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| Abstract Factory | 抽象工厂模式 | Creational | Create families of related or dependent objects without specifying their concrete classes. | host renderers; install-target adapters; manifest templates | Plausible | gstack Host renderers and ECC install-target adapters are candidates for compatible Host product families. | Generating one Host-specific file does not establish a family of compatible products. | No |
| Builder | 建造者模式 | Creational | Separate construction of a complex object from its representation so one process can create different representations. | generator Skill; construction schema; profile-specific writers | Strong | dot-skill stages material collection, analysis, schema-driven writing, and installation of generated Skillware. | A single template expansion without staged construction or representation variation is not Builder. | No |
| Factory Method | 工厂方法模式 | Creational | Define an object-creation interface while allowing concrete creators to choose the product produced. | creator Skill; family profiles; scaffolding scripts | Plausible | dot-skill family profiles are candidate concrete creators. | Selecting a static template without a creator contract or product variation is not Factory Method. | No |
| Prototype | 原型模式 | Creational | Create new objects by copying a prototypical instance. | Skill templates; profile seeds; controlled copy scripts | Plausible | dot-skill templates and profile seeds are candidate prototypes; controlled copy semantics require further validation. | Copying a directory without declared identity and customization semantics is not Prototype. | No |
| Singleton | 单例模式 | Creational | Ensure a class has one instance and provide a global access point to it. | session-scoped service; profile configuration owner | Weak | No focused case demonstrates a unique runtime instance across Host and session boundaries. | A conventional root file or preferred entry point does not prove runtime instance uniqueness. | No |
| Adapter | 适配器模式 | Structural | Convert one interface into another expected by clients so incompatible collaborators can work together. | canonical Skill; Host binding; manifest; tool mapping | Strong | gstack translates canonical Skill behavior into Host-specific paths, metadata, and tool vocabulary. | Renaming or copying a file without semantic interface translation is not Adapter. | Yes |
| Bridge | 桥接模式 | Structural | Separate an abstraction from its implementation so both can vary independently. | canonical action vocabulary; Host resolver; tool mapping | Plausible | gstack's canonical actions and Host resolvers are a candidate separation of abstraction and implementation dimensions. | A translation shim is Adapter unless abstraction and implementation are independently variable dimensions. | No |
| Composite | 组合模式 | Structural | Compose objects into part-whole trees so clients treat individual objects and compositions uniformly. | parent Skill; child Skills; workflow manifest | Strong | OpenMontage's stage system is a candidate; the companion repository verifies a common component contract and acyclic part-whole structure. | A directory of unrelated Skills without a shared component contract is not Composite. | Yes |
| Decorator | 装饰模式 | Structural | Add responsibilities to an object dynamically while preserving its component interface. | wrapper Skill; activation wrapper; pre-hook; post-hook | Strong | Caveman activation wrappers are a candidate; the companion repository verifies contract-preserving responsibility layering. | A hook that changes the required task contract or replaces core behavior is not Decorator. | Yes |
| Facade | 外观模式 | Structural | Provide a unified interface to subsystem interfaces so the subsystem is easier to use. | entry Skill; root router; invocation policy | Strong | Superpowers using-superpowers and gstack's root router implement unified entry contracts over independently addressable specialist Skills. | A README or catalog that only lists Skills supplies navigation, not an executable access contract. | Yes |
| Flyweight | 享元模式 | Structural | Share fine-grained objects efficiently by separating reusable intrinsic state from contextual extrinsic state. | shared references; shared templates; caches | Weak | Shared references alone do not demonstrate intrinsic state sharing and separated extrinsic state. | Ordinary file reuse or deduplication does not establish the intrinsic and extrinsic state split. | No |
| Proxy | 代理模式 | Structural | Provide a surrogate that controls access to another object through a compatible interface. | local daemon; MCP server; permission gateway | Plausible | gstack's browser daemon and EvoZeus dispatch paths are audit candidates; neither currently demonstrates a verified same-interface surrogate relation. | A gateway or protocol adapter without a compatible surrogate interface is not Proxy. | No |
| Chain of Responsibility | 职责链模式 | Behavioral | Pass a request along potential handlers until one handles it, avoiding a fixed sender-receiver coupling. | handler Skills; typed handoff records; workflow routes | Plausible | ECC and financial-services orchestration paths are candidates; each handler must independently decide to handle or forward the same request. | A fixed processing pipeline in which every stage runs is not Chain of Responsibility. | No |
| Command | 命令模式 | Behavioral | Encapsulate a request as an object so invocation, queuing, logging, and undo can vary independently. | command record; slash command; receiver Skill; audit record | Plausible | ECC command-to-Skill routes are candidates; invoker, request object, receiver, and audit or undo semantics require verification. | A shortcut or CLI command without an encapsulated request and receiver relationship is not Command. | No |
| Interpreter | 解释器模式 | Behavioral | Represent a language grammar and interpret sentences expressed in that language. | workflow grammar; parser; expression records | Plausible | OpenMontage's pipeline language and parser are candidates when grammar and expression participants are evidenced. | General model interpretation of prose without an explicit grammar and expression structure is not Interpreter. | No |
| Iterator | 迭代器模式 | Behavioral | Access aggregate elements sequentially without exposing the aggregate's underlying representation. | catalog traversal contract; paginated scanner | Weak | Catalog scans and ordinary loops provide insufficient evidence of an independent traversal contract. | A loop or recursive file scan alone does not establish an Iterator participant contract. | No |
| Mediator | 中介者模式 | Behavioral | Encapsulate peer interactions so colleagues do not refer to each other explicitly. | coordinator Skill; parent Agent; coordination manifest | Strong | financial-services workflow orchestration and EvoZeus coordination are candidates; the companion repository verifies colleague isolation. | A central God Skill that absorbs specialist work is not a sound Mediator. | Yes |
| Memento | 备忘录模式 | Behavioral | Capture internal state without breaking encapsulation so the object can later be restored. | release snapshot; checkpoint; caretaker harness | Strong | SkillOpt backups and EvoZeus release snapshots provide partial evidence of capture, caretaker ownership, and recovery. | A backup or Git commit without an owned operational restore path is not necessarily Memento. | Yes |
| Observer | 观察者模式 | Behavioral | Define a one-to-many dependency that notifies observers when a subject changes state. | lifecycle event; hook; subscription record; event consumer | Strong | ECC hooks and OpenMontage lifecycle events are candidates; explicit subject, registration, and notification semantics require path validation. | Polling or an unregistered hook without a subject notification contract is not Observer. | Yes |
| State | 状态模式 | Behavioral | Let an object alter its behavior when its internal state changes. | persisted checkpoint; state record; transition manifest | Strong | SkillOpt stages and OpenMontage checkpoints provide candidate persisted state-dependent behavior. | Prose phase names or stateless routing do not establish behavior controlled by explicit state. | Yes |
| Strategy | 策略模式 | Behavioral | Define interchangeable algorithms behind a common interface so they vary independently of their context. | specialist Skills; selection policy; shared request contract | Strong | ui-ux-pro-max retrieval procedures and financial-analysis workflows provide candidate interchangeable strategy families. | Alternative subcommands with incompatible result contracts are not interchangeable Strategies. | Yes |
| Template Method | 模板方法模式 | Behavioral | Fix an algorithm skeleton while allowing subclasses to redefine selected steps without changing its sequence. | workflow Skill; invariant skeleton; domain profiles | Strong | Superpowers workflows provide candidate invariant skeletons with bounded task-specific steps. | A freely reordered checklist or runtime-swappable whole algorithm is not Template Method. | Yes |
| Visitor | 访问者模式 | Behavioral | Represent operations over an object structure separately so new operations do not change element classes. | repository scanner; validator; heterogeneous element records | Plausible | Repository scanners and validators are candidates when operation dispatch over a stable heterogeneous element structure is evidenced. | A generic recursive scan without element-specific dispatch and a stable structure is not Visitor. | No |

The current screening distribution is 11 Strong, 9 Plausible, and 3 Weak. Only the 10 rows marked **Yes** have detailed GoF samples in this repository plan; the other 13 do not claim one.
