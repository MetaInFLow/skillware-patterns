# Why this is not Mediator

The peer mesh requires every specialist to know every other specialist. Adding
or changing one participant changes multiple callers, and a peer failure can
propagate across the mesh. Four participants create twelve directed peer
relationships instead of four central report relationships.

The fallback God Skill is not a correction: by repeating build, security,
documentation, and approval work, it erases Colleague ownership rather than
encapsulating interaction. A sound Mediator owns communication and joint
policy; it does not absorb specialist responsibilities.

同伴网状结构造成多方耦合；God Skill 吸收专长职责。两者都不符合中介者模式。
