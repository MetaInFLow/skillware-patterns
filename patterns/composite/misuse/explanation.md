# Why this is not Composite

The neighboring `SKILL.md` is a directory of related investment capabilities,
but relation by topic is not part-whole composition. Its leaves return a list,
different objects, and an unstructured document, while the root returns paths.
A Client therefore cannot invoke an atomic result and a grouped result through
one Component contract.

A workflow also fails Composite when it merely sequences those incompatible
shapes. Composite requires uniform Leaf and Composite treatment plus explicit,
valid part-whole references. Folder nesting, a shared investment theme, or a
workflow label does not supply either property. The constructive sample proves
the distinction with the same five result keys at every node, empty Leaf
children, recursive Composite children, and cycle and reference validation.
