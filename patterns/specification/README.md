# Specification / 规约模式

This standalone record transfers Eric Evans's Domain-Driven Design
Specification pattern to an Expense Approval Policy Skillware Unit. It is a
domain pattern, explicitly not a Gang of Four pattern.

The constructive sample contains immutable named Specifications, an exact
Candidate contract, reusable AND/OR/NOT policies, structured explanations,
fixtures, and a standard-library executable oracle. Start with
[`definition.md`](definition.md) and [`sample/README.md`](sample/README.md).

## Upstream Skill example

No public upstream Skill was admitted as a bounded Specification correspondence
in this release; the controlled status is **not observable**. The local
expense-policy sample is constructive evidence only.
